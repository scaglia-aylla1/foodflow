# ADR-005 - Arquitetura de Microsserviços

## Status

Accepted

## Contexto

O objetivo do projeto é demonstrar competências exigidas pelo mercado para vagas Java AWS.

## Alternativas Consideradas

### Opção 1

Monólito modular.

### Opção 2

Microsserviços.

## Decisão

Utilizar arquitetura de microsserviços.

## Consequências

### Positivas

- Demonstração de arquitetura distribuída
- Uso de Kafka
- Uso de ECS
- Uso de API Gateway

### Negativas

- Maior complexidade operacional
